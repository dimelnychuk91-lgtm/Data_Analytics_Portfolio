# python-project
Exploratory Data Analysis for Online Store
Бізнес-мета проєкту
Цей проєкт присвячений аналізу міжнародних продажів онлайн-магазину. Основна ціль полягає у пошуку точок росту, оцінці ефективності каналів збуту (Online vs Offline), виявленні сезонних патернів та оптимізації товарного портфеля за допомогою ABC-аналізу.

Стек технологій
Python

Pandas (обробка та агрегація даних)

Matplotlib та Seaborn (візуалізація)

Jupyter Notebook / Google Colab

Головні інсайти.
Специфіка бізнес-моделі (B2B патерни): Аналіз динаміки продажів вказує на те, що компанія працює переважно у форматі великого опту. Наприклад, у 2012 році зафіксовано аномальний стрибок прибутку (+24%), а в розрізі країн немає стабільного лідера (географія постійно змінюється). Бізнес тримається на разових великих угодах, а не на лояльній базі роздрібних клієнтів.

Логістика та час відвантаження: Медіанний час відвантаження становить 25 днів і є абсолютно ідентичним для всіх категорій товарів та регіонів світу. Логістична система не має "пріоритетних" напрямків.

Кореляція між прибутком та доставкою: Виявлено майже нульову кореляцію між сумою прибутку та часом, витраченим на відвантаження. Фінансові показники не залежать від швидкості логістики, що є нетиповим для роздрібної торгівлі і ще раз підтверджує оптовий характер угод.

Специфічна бізнес-сезонність: Пік продажів припадає на червень (закриття піврічних контрактів). Також виявлено чіткі патерни для окремих категорій: Офісні товари мають пік у лютому (початок фінансового року в багатьох компаніях), а М'ясо — у листопаді (підготовка до свят). Попит диктується циклами закупівель, а не погодою.

Ефективність каналів (Online vs Offline): Розподіл медіанного прибутку з одного замовлення та розкид сум практично ідентичні для обох каналів. Вони працюють однаково ефективно і залучають клієнтів зі схожою платоспроможністю.

ABC-аналіз товарного портфеля: Основну частку прибутку (80%) формують такі категорії, як Cosmetics, Office Supplies та Beverages. Це стабільна база бізнесу, на якій потрібно фокусувати складські запаси.

Структура даних
У проєкті використовуються три пов'язані датасети:

events.csv: інформація про транзакції, дати замовлень, канали продажу та фінансові показники.

products.csv: довідник категорій товарів.

countries.csv: географічні дані по регіонах.
